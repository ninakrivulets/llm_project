# Отчет HW2

Для MVP мы выбрали стек: React + TypeScript для фронтенда, FastAPI на Python для бэкенда и Qdrant для векторного поиска.
Это привычные нам инструменты + быстро позволяют собрать прототип.
От гибридного поиска решили отказаться, потому что на имеющемся сервере он давал бы лишнюю нагрузку, а прирост качества на русском языке в наших условиях оказался невысоким.

Основной принцип разработки на этом этапе был прагматичным: делать как можно быстрее, чтобы как можно раньше получить рабочий MVP и проверить базовый сценарий использования.
Поэтому мы сознательно выбирали более простые и понятные решения вместо избыточно сложной архитектуры, а также ограничивали функциональность тем, что необходимо для демонстрации core-функционала проекта.

Основные сложности возникли на этапе парсинга документов и подготовки данных к загрузке.
Чтобы не тратить слишком много времени на построение отдельного сложного пайплайна, мы решили вести разбор и первичную обработку данных в `ipynb`-ноутбуках,
а саму загрузку в систему вынести в отдельный скрипт.
Это позволило разделить этап экспериментов с парсингом и этап загрузки данных, упростить отладку и сохранить гибкость для дальнейших доработок.
